# VPI-Bench 论文总结

**论文标题**: VPI-Bench: Visual Prompt Injection Attacks for Computer-Use Agents

**作者**: Tri Cao, Bennett Lim, Yue Liu, Yuan Sui, Yuexin Li, Shumin Deng, Lin Lu, Nay Oo, Shuicheng Yan, Bryan Hooi

**机构**: National University of Singapore, Cyber Emerging Tech and R&D

**发表**: ICLR 2026

---

## 1. 摘要

本文提出了**VPI-Bench**，一个用于评估**可视化提示注入（Visual Prompt Injection, VPI）**攻击计算机代理（Computer-Use Agents, CUAs）的基准测试。

**核心成果**：
- 提出端到端威胁模型，VPI在黑盒设置下操纵CUAs执行未授权操作或泄露敏感信息
- 提出VPI-Bench基准测试，包含**306个测试用例**，涵盖5个广泛使用的平台
- 当前CUAs和BUAs在某些平台上可被欺骗率高达 **51%** 和 **100%**
- 现有防御方法仅提供有限的改进
- 强调了需要鲁棒、上下文感知的防御来确保多模态AI代理在现实环境中的安全部署

---

## 2. 创新点

### 2.1 可视化提示注入（VPI）攻击

与传统基于HTML的注入不同，VPI攻击：
- 渲染在屏幕上，可被基于视觉的CUAs感知
- 同时存在于HTML中，使基于HTML的BUAs也能访问
- 即使人类容易检测，但在用户委托任务给代理无监督的情况下构成威胁

### 2.2 端到端威胁模型

四组件相互依赖的威胁模型：

| 组件 | 说明 |
|------|------|
| 良性用户提示 (Tbenign) | 用户描述的无害任务 |
| Web平台 (w) | 伪真实网页（如amazon.com） |
| 视觉攻击提示 (vadv) | 注入的恶意视觉内容 |
| 执行环境 (E) | 沙箱环境，模拟本地和互联网资源 |

### 2.3 VPI-Bench基准测试

- **306个测试用例**覆盖5个平台
- 每个测试用例是网页平台的变体
- 设计为交互式，部署在现实环境中
- 包含视觉嵌入的恶意提示

### 2.4 多维度评估框架

| 维度 | 说明 |
|------|------|
| 尝试率 (AR) | 代理尝试恶意行为的比例 |
| 成功率 (SR) | 代理成功完成恶意任务的比例 |

---

## 3. 核心原理/框架

### 3.1 VPI攻击流程

```
[用户委托任务] → [CUA与网页交互] → [遇到恶意视觉内容]
                                            ↓
                        [被操纵执行恶意任务Tmal]
                                            ↓
                        [未授权操作/数据泄露]
```

### 3.2 攻击场景覆盖

| 平台 | 领域 | 攻击类型 |
|------|------|---------|
| Amazon | 购物 | 未授权操作 |
| Booking.com | 旅游 | 未授权操作 |
| BBC News | 新闻 | 未授权操作 |
| Email | 邮件 | 隐私泄露 |
| Messenger | 即时通讯 | 隐私泄露 |

### 3.3 恶意任务分类

| 攻击目标 | 描述 |
|----------|------|
| 未授权操作 | 文件修改、命令执行、发送未授权消息 |
| 隐私泄露 | 敏感数据外泄到外部 |

### 3.4 评估方法

使用三个独立的前沿LLM进行多数投票判断：
- 判断恶意任务是否被尝试
- 判断恶意任务是否成功完成
- 至少2/3 LLM返回正面决策才认定为尝试/成功

## 4. 基准对比对象和方法

### 4.1 测试的代理类型

| 类型 | 说明 |
|------|------|
| CUAs | 计算机使用代理，具有完整系统访问权限 |
| BUAs | 浏览器使用代理，仅在Web环境操作 |

### 4.2 代表性代理

| 代理 | 特点 |
|------|------|
| Anthropic's CUA | 通过视觉感知进行完整计算机控制 |
| UI-TARS | 端到端微调的GUI代理，直接在截图上操作 |
| OpenAI系统 | 仅支持有限的基于Web的操作 |

### 4.3 基线防御方法

| 防御方法 | 说明 |
|----------|------|
| 现有VPI防御 | 测试其效果 |

### 4.4 评估指标

$$AR = \frac{N_{attempted}}{N}, \quad SR = \frac{N_{successful}}{N}$$

## 5. 实验结果

### 5.1 主要攻击效果

| 代理类型 | 某些平台欺骗率 |
|----------|---------------|
| CUAs | 最高 **51%** |
| BUAs | 最高 **100%** |

### 5.2 跨平台结果

| 平台 | 攻击效果 |
|------|---------|
| Amazon | 高欺骗率 |
| Booking.com | 高欺骗率 |
| BBC News | 中等欺骗率 |
| Email | 高隐私泄露率 |
| Messenger | 高隐私泄露率 |

### 5.3 防御方法效果

| 防御方法 | 改进效果 |
|----------|---------|
| 现有防御 | **仅有限改进** |

### 5.4 关键发现

1. **视觉攻击有效**: VPI攻击可成功欺骗视觉基础的CUAs
2. **HTML仍可访问**: 恶意内容同时存在于HTML中，影响BUAs
3. **级联行为**: 攻击可导致多步骤恶意行为链
4. **现实威胁**: 代理可能被诱导执行未授权操作或泄露私人数据

---

## 6. 结论

### 6.1 主要贡献

1. **端到端VPI威胁模型**: 首个针对CUAs的VPI黑盒攻击威胁模型
2. **VPI-Bench基准**: 306个测试用例，覆盖多种平台和攻击场景
3. **系统性评估**: 对CUAs和BUAs的全面鲁棒性评估
4. **安全启示**: 强调需要鲁棒、上下文感知的防御

### 6.2 安全启示

| 发现 | 启示 |
|------|------|
| 高欺骗率 | 现有代理存在严重安全漏洞 |
| 有限防御 | 需要更先进的防御技术 |
| 级联攻击 | 需要端到端安全评估 |

### 6.3 研究局限性

- 合成网页环境可能不完全代表真实世界
- 沙箱环境与真实部署环境可能有差异
- 需要持续更新以跟上代理能力的发展
