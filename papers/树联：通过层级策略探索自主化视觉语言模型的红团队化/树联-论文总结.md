# 树联 (TreeTeaming) 论文总结

**论文标题**: TreeTeaming: Autonomous Red-Teaming of Vision-Language Models via Hierarchical Strategy Exploration

**作者**: Chunxiao Li, Lijun Li, Jing Shao

**机构**: Shanghai Artificial Intelligence Laboratory

**发表**: ICLR 2026

---

## 1. 摘要

本文提出了**TreeTeaming**，一个自动化红队测试框架，通过**层级策略探索**来发现视觉语言模型（VLMs）的安全漏洞。

**核心成果**：
- 在12个主流VLMs上进行了大规模实验
- 在11个模型上达到**最先进的攻击成功率**
- GPT-4o上达到 **87.60%** 的攻击成功率
- 生成的攻击具有**平均23.09%的毒性降低**，展示更好的隐蔽性
- 策略多样性超越所有先前公开的越狱策略集合

---

## 2. 创新点

### 2.1 从线性探索到动态策略树

传统方法的局限：
- 依赖预定义策略集
- 无法发现新颖、多样的攻击方式

TreeTeaming的突破：
- 将策略探索从静态测试重新定义为**动态进化过程**
- 构建和扩展**策略树（Strategy Tree）**
- 自主决定是进化有前景的攻击路径还是探索多样的策略分支

### 2.2 三核心组件架构

| 组件 | 功能 |
|------|------|
| **策略树 & Orchestrator** | 指导策略进化 |
| **多模态执行器 (Actuator)** | 执行复杂策略 |
| **策略一致性检查器** | 验证最终样本与攻击意图一致 |

### 2.3 层级策略树结构

```
        [Root: 终极目标]
              |
    ┌─────────┴─────────┐
[策略类别1] [策略类别2] [策略类别3]
    |           |           |
[具体策略] [具体策略] [具体策略]
```

- **根节点**: 封装红队测试的最终目标
- **父节点（策略类别）**: 分组相关攻击方法（如"认知偏见利用"、"权威 impersonation"）
- **叶节点（可执行策略）**: Actuator可执行的具体策略描述

### 2.4 探索-利用平衡机制

动态阈值τ控制探索与利用的平衡：
- **Exploitation（利用）**: 优化高ASR策略
- **Exploration（探索）**: 发现新攻击向量
- τdynamic 实现平滑过渡

---

## 3. 核心原理/框架

### 3.1 系统架构

```
[策略树] ←→ [Orchestrator]
                ↓
    [执行/探索决策]
                ↓
[Actuator] ←→ [Judge Model]
                ↓
        [目标VLM]
```

### 3.2 执行器工具套件

Actuator配备了**11个工具**来执行复杂策略：
- 将策略转化为图文测试样本
- 多工具协同完成复杂攻击

### 3.3 失败原因分析模型

提供系统级学习的反馈：
- 收集样本失败日志
- 统计分析和归纳
- 识别主导失败模式
- 为Orchestrator的下一步决策提供输入

### 3.4 策略级别归属循环

Macro级学习循环：
1. 叶节点所有测试尝试完成后
2. 收集相关样本失败日志
3. 识别该子策略的主导失败模式
4. 记录到策略树的对应叶节点

## 6. 结论

### 6.1 主要贡献

1. **新范式**: 引入自动化漏洞发现的层级策略探索新范式
2. **动态策略树**: 将静态测试转变为动态进化发现过程
3. **高效执行**: 多模态Actuator + 策略一致性检查确保测试有效性
4. **全面评估**: 12个VLMs大规模实验验证

### 6.2 安全启示

- 静态启发式方法存在根本局限
- 需要主动探索而非被动测试
- 前沿AI模型需要系统性红队测试
