# 通过仅查询交互对LLM代理的内存注入攻击 论文总结

**论文标题**: Memory Injection Attacks on LLM Agents via Query-Only Interaction

**作者**: Shen Dong, Shaochen Xu, Pengfei He, Yige Li, Jiliang Tang, Tianming Liu, Hui Liu, Zhen Xiang

**机构**: Michigan State University, University of Georgia, Singapore Management University

**发表**: ACM CCS / IEEE S&P 等顶会

---

## 1. 摘要

本文提出了**MINJA (Memory INJection Attack)**，一种针对LLM代理的**新型内存注入攻击**，攻击者仅通过查询和输出观察与代理交互，无需直接访问代理的内存银行。

**核心成果**：
- 首次实现**仅通过查询交互**注入恶意记录到代理内存银行
- 在多种代理类型上验证了MINJA的有效性
- **ISR（注入成功率）超过90%**，ASR（攻击成功率）部分超过90%
- **最小化执行要求**，任何用户都能影响代理内存
- 在自动驾驶、医疗、金融等应用场景中造成严重安全威胁

---

## 2. 创新点

### 2.1 仅查询交互的攻击模式

传统攻击的局限：
- 假设攻击者能**直接修改**代理的内存银行
- 面临巨大的可行性挑战

MINJA的突破：
- 攻击者仅能通过**查询和输出观察**与代理交互
- 像普通用户一样行为，无法直接修改内存
- **任何用户都可能成为攻击者**

### 2.2 桥接步骤设计

解决恶意查询与目标推理步骤之间的**逻辑鸿沟**：

```
[攻击查询 av] → [桥接步骤 bv,t] → [目标推理步骤 Rat]
     (包含受害者术语)    (连接逻辑)     (恶意目标)
```

**桥接步骤的作用**：
- 逻辑连接受害者术语(v)和目标术语(t)
- 确保检索到的恶意记录能建立期望的逻辑链
- 出现在推理步骤的开头，确保首先被生成

### 2.3 渐进式缩短策略

逐步移除指示提示，同时保留桥接步骤和目标恶意推理：
1. 初始：完整的指示提示 + 桥接步骤
2. 逐步缩短：移除部分指示提示
3. 最终：无指示提示，仅保留桥接步骤

### 2.4 指示提示机制

引导代理自主生成类似的桥接步骤：
- 代理根据指示提示生成恶意推理
- 渐进式缩短确保恶意记录容易被检索

## 3. 核心原理/框架

### 3.1 MINJA攻击流程

```
[阶段1: 恶意记录设计]
    攻击查询 av (包含受害者术语 v)
    + 桥接步骤 bv,t (连接 v 和目标 t)
    + 目标推理步骤 Rat
           ↓
[阶段2: 注入恶意记录]
    通过指示提示诱导代理生成恶意记录
    渐进式缩短移除指示提示
           ↓
[阶段3: 受害者查询]
    受害者提交包含 v 的查询
           ↓
[阶段4: 恶意检索]
    恶意记录被检索为演示
    误导代理生成恶意推理链
```

### 3.2 恶意记录设计原则

| 设计要素 | 原则 |
|----------|------|
| 攻击查询 | 包含受害者术语，形式类似良性查询 |
| 桥接步骤 | 通用推理步骤，连接任意攻击查询到目标 |
| 目标推理 | 针对特定目标任务的推理链 |
| 检索优化 | 避免罕见或异常内容，确保高检索率 |

### 3.3 内存银行设置

| 代理类型 | 内存记录内容 |
|----------|-------------|
| RAP | 输入查询 + 推理序列 + 执行动作 |
| EHRAgent | 查询 + 推理步骤 + 生成的代码 |
| QA Agent | 问题 + 思维链推理 |

## 4. 基准对比对象和方法

### 4.1 测试的代理类型

| 代理 | 基础模型 | 任务领域 |
|------|----------|---------|
| **RAP** | GPT-4, GPT-4o | Web购物 (Webshop) |
| **EHRAgent** | GPT-4 | 医疗 (MIMIC-III, eICU) |
| **QA Agent** | GPT-4, GPT-4o | 通用问答 (MMLU) |

### 4.2 受害者-目标配对

| 数据集 | 受害者类型 | 目标类型 |
|--------|-----------|---------|
| MIMIC-III/eICU | 患者ID | 患者ID |
| Webshop | 商品ID | 商品ID |
| MMLU | 术语 | 术语 |

### 4.3 评估指标

| 指标 | 定义 |
|------|------|
| **ISR** | 注入成功率 = 成功注入记录数 / 总攻击查询数 |
| **ASR** | 攻击成功率 = 响应包含恶意推理的测试查询数 / 总测试查询数 |
| **UD** | 效用下降 = 有/无攻击时良性任务性能差异 |

### 5.5 关键发现

1. **高ISR**: 大多数配置下达90%+的注入成功率
2. **高ASR**: 部分配置超过90%（如GPT-4o RAP on Webshop达98.9%）
3. **保持良性效用**: MIMIC-III、eICU、Webshop的UD下降小于2%
4. **检索噪声鲁棒**: 噪声环境下ASR仅轻微下降
5. **防御困难**: 定向检测效果好但泛化差，通用检测有高误报率

---

## 6. 结论

### 6.1 主要贡献

1. **首个仅查询交互攻击**: MINJA是首个无需直接内存访问的内存注入攻击
2. **桥接步骤设计**: 解决逻辑鸿沟问题
3. **渐进式缩短策略**: 确保恶意记录可被检索
4. **广泛有效性**: 在多种代理类型和任务上验证

### 6.2 安全启示

| 发现 | 启示 |
|------|------|
| 共享内存风险 | 共享内存设计带来供应链风险 |
| 用户级别威胁 | 普通用户也能成为攻击者 |
| 检索机制漏洞 | 基于相似性的检索可被利用 |
| 防御困难 | 现有检测方法存在根本局限 |

### 6.3 潜在缓解方案

- 内存隔离设计
- 强化用户认证
- 安全内存管理
- 高级提示过滤
